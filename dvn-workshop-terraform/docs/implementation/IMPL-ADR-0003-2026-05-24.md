# IMPL-ADR-0003-2026-05-24

## Reference ADR
ADR-0003: EKS Cluster com Managed Node Group para Workloads de Container

## Summary
Implemented the Terraform stack `02-eks-stack-ai` that provisions an Amazon EKS cluster (Kubernetes 1.31) with a managed node group (2x t3.medium ON_DEMAND) in the private subnets of the existing VPC provisioned by stack `01-networking-stack-ai`. All five control plane log types are enabled. Three managed addons (vpc-cni, coredns, kube-proxy) are configured. State is stored in the shared S3 backend under key `eks/terraform.tfstate`.

## Files Created

| File | Purpose |
|---|---|
| `02-eks-stack-ai/versions.tf` | Terraform block, S3 backend (key: `eks/terraform.tfstate`), provider `hashicorp/aws ~> 6.0` |
