## Design Decisions

### `count` vs `for_each`
Used `count` throughout as the input variables are ordered lists of CIDRs and AZs.
`for_each` would require converting to maps; `count` is cleaner here and aligns with the ADR guidance.

### Flow Logs conditionality
All four flow-log resources use `count = var.enable_flow_logs ? 1 : 0` so the feature
can be toggled without destroying unrelated resources.

### IAM Role Policy scope
The `logs:PutLogEvents` and related actions are scoped to the specific CloudWatch Log Group ARN
with a `:*` suffix (to cover log streams), satisfying least-privilege.

### NAT Gateway `depends_on`
Explicit `depends_on = [aws_internet_gateway.main, aws_subnet.public]` added to each
NAT Gateway to ensure the IGW is attached and public subnets are created before the
NAT GW allocation is attempted. The ADR mandates this.

### `default_tags` on provider
All mandatory tags (`Environment`, `Project`, `ManagedBy`, `Stack`) are injected at the
provider level via `default_tags`. Individual resources only define `Name` and
resource-specific tags to avoid duplication.

## Rollback Plan

This is a greenfield stack with no existing state. To roll back:

1. `terraform destroy` — requires explicit human confirmation.
2. No stateful resources (RDS, S3 with objects) are present in this stack, so destroy is safe.
3. NAT Gateway EIPs will be released; confirm no other resources reference them before destroy.
